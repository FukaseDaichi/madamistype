# アンケート傾向ページ 設計方針（暫定）

## 1. 文書の目的

本書は、[`results.md`](./results.md) に整理されたアンケート集計をサイト掲載用の公開ページへ落とし込むための暫定設計方針である。  
今回は実装前の方針整理を目的とし、現行 UI と導線の変更点、ページ構成、未確定事項をまとめる。

参照:

- [specification.md](./specification.md)
- [ui-design-spec.md](./ui-design-spec.md)
- [frontend-directory-structure-spec.md](./frontend-directory-structure-spec.md)
- [results.md](./results.md)

## 2. 背景

- `docs/results.md` には、206 件のアンケート回答から見えたタイプ分布、作品推薦傾向、自由記述の所感がまとまっている
- この内容を、サイト上で読める常設の公開ページとして掲載したい
- あわせてトップページからの導線を追加し、診断以外にも「傾向を読む」入口を用意したい
- 現在の共有結果ページには Google フォームへの「おすすめマダミス」導線があるが、今後はこの外部フォーム導線をなくす

## 3. 今回の改修方針

今回の方針では、次の 3 点をセットで扱う。

1. アンケート結果を紹介する新規公開ページを追加する
2. トップページの診断開始導線の横に「傾向を見る」ボタンを追加する
3. 共有結果ページの「おすすめマダミス」導線を廃止し、その位置を「自身のキャラ記載用の場所」として再定義する

## 4. 新規ページの位置づけ

### 4.1 役割

新規ページは、診断そのものではなく「診断ユーザー全体の傾向を読む記事ページ」とする。  
個別の診断結果ページとは分けて、集計記事として読ませる。

### 4.2 想定ルート

暫定の公開ルートは `/trends` とする。

採用理由:

- `/results` だと個人の診断結果ページと意味が近く、導線上で混同しやすい
- ボタン文言「傾向を見る」と対応が取りやすい
- 公開記事として短く扱いやすい

### 4.3 検索エンジン向け扱い

- index 対象
- canonical は `/trends`
- OGP は既存のサイトトーンに合わせた静的画像を流用するか、必要なら後続で専用画像を追加する

## 5. ページ内容の設計

### 5.1 記事のトーン

- `docs/results.md` のやわらかい語り口をベースにする
- ただしサイト掲載用として、見出し・段落・箇条書きを整理して読みやすく再構成する
- 「協力的な方々の分布であり、全体傾向そのものではない」という注意書きは冒頭で明示する

### 5.2 記事構成

新規ページは、概ね次の順で構成する。

1. 導入
2. 全体の傾向
3. 全体でよく名前が挙がった作品
4. タイプ別・よく薦められていた作品
5. 自由記述で多かった意見
6. まとめ

### 5.3 冒頭で強調する要点

冒頭要約では、少なくとも次を見せる。

- 回答数は 206 件
- 多かったタイプは `TFEI`、`TREI`、`OFEI`
- 全体でよく名前が挙がった作品は「この闇をあなたと」「ふわふわクリームさらにジューシー」「シュレディンガーの密室」「鬼哭館の殺人事件」「エイダ」

### 5.4 タイプ別紹介の見せ方

タイプ別紹介は、`results.md` の内容をそのまま長文で並べるだけではなく、次の粒度で整理する。

- タイプ名 + タイプコード
- 短い傾向要約
- 代表的なおすすめ作品

回答数の多いタイプと、それ以外のタイプでセクションを分ける案を採用する。

## 6. UI / レイアウト方針

### 6.1 デザインの方向性

- 既存のトップページ・タイプ詳細ページと同じく、ケースファイル風の世界観に寄せる
- ただし個別タイプページではなく、記事ページとして本文の可読性を優先する
- 長文を読むページなので、本文カラム幅・見出し階層・余白設計をトップページより落ち着かせる

### 6.2 必要な UI 要素

- ヘッダーまたはパンくず相当の戻り導線
- 記事タイトル
- 概要リード
- セクション見出し
- 作品名やタイプ名を見やすくする簡易カードまたは箇条書き
- ページ下部の診断導線

### 6.3 診断への返し導線

記事の最後には「自分も診断してみる」導線を置く。  
記事を読んだあとに `/` または `/diagnosis` へ戻れる導線を確保する。

## 7. トップページの変更方針

### 7.1 変更箇所

トップページでは、診断開始フォームの主ボタン付近に「傾向を見る」ボタンを追加する。

### 7.2 導線ルール

- 主ボタンは従来どおり診断開始
- その横に副導線として「傾向を見る」を置く
- 「傾向を見る」は新規ページ `/trends` へリンクする
- モバイルでは縦積み、横幅が確保できる画面では横並びを許容する

### 7.3 意図

- まだ診断する前の人にも、サービス内容の理解材料を増やす
- 「どんなタイプが多いのか」「どんな作品が薦められているのか」を先に読めるようにする
- 診断 CTA 一択だったトップ導線に、情報閲覧の入口を追加する

## 8. 共有結果ページの変更方針

### 8.1 廃止するもの

現行の共有結果ページで、診断直後にだけ表示している以下の導線は廃止する。

- 「おすすめマダミス」ボタン
- Google フォームへの外部リンク
- フォーム投稿を促す補足文

### 8.2 置き換えるもの

上記の導線があった位置は、「そのページの自身のキャラについて記載する場所」として扱う。

本方針書では、まず次のように定義する。

- 共有結果ページ内の診断直後専用アクション領域の二次導線枠を転用する
- 外部フォームではなく、結果ページの文脈の中で完結する表現にする
- 初回実装では、少なくとも「ここは自分のキャラ情報を書く／載せるための場所」という意味づけを持たせる

### 8.3 この時点で未確定のこと

「自身のキャラの記載」の具体仕様は、今回の方針整理時点では未確定とする。

未確定項目:

- ただの説明文にするか
- 固定テキスト枠にするか
- ユーザー入力を保持する機能にするか
- 共有 URL に含めるか、ローカル保存にするか

そのため、実装時はまず「フォームリンクを消し、キャラ記載用の場所として差し替える」までを第一段階とし、入力・保存の仕様は後続で分離して詰める。

## 9. 実装構成の想定

現行の App Router 構成に合わせ、次のような追加・変更を想定する。

### 9.1 新規追加

- `app/(marketing)/trends/page.tsx`
- `components/trends/` 配下のページ本文コンポーネント群

### 9.2 既存変更

- `components/diagnosis/start-diagnosis-form/start-diagnosis-form.tsx`
- `components/type/type-detail-page-content/type-detail-hero-section.tsx`

### 9.3 データの持ち方

`docs/results.md` は設計・原稿の元資料として扱い、実際の表示データはアプリ側で扱いやすい形に再構成する。

理由:

- 現行構成には MD/MDX を直接公開ページへ流し込む仕組みがない
- 既存サイトは `app/` + `components/` 中心の静的構成で揃っている
- 記事ページでも見出しや強調表現を UI として調整しやすくなる

## 10. 今回の前提解釈

この設計方針では、ユーザー要望のうち次を前提として解釈する。

- 「傾向を見る」は、アンケート集計記事ページへの公開導線である
- 追加場所はトップページの診断開始ボタン付近である
- 「結果ページの『おすすめマダミス』のところ」は、共有結果ページの診断直後専用アクション領域を指す
- 「自身のキャラの記載についての場所とする」は、外部フォーム導線をやめて、そのページ内でキャラ情報を扱うための枠へ転用する意味として扱う
