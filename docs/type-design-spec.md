# マダミスタイプ診断 タイプ設計仕様書

## 1. 文書目的

本書は [specification.md](./specification.md) の「5. タイプ設計」を具体化し、正式なタイプコードとタイプ名称を定義するための補助仕様書である。

本書で定義する内容は以下の通りとする。

- 4軸16タイプのタイプコード規則
- 16タイプの正式名称
- タイプ名称の命名方針
- タイプマスタに登録する初期値

## 2. 設計方針

### 2.1 基本方針

- タイプコードは4文字固定とし、1文字ごとに1つの診断軸へ対応させる
- コード順は診断ロジックと同じく「発言/観察 → 事実/推理 → 論理/感情 → 計画/即興」の順に固定する
- タイプ名称は性格ラベルではなく、マーダーミステリー卓上での立ち回りを想起できる役割名とする
- 名称は拡散しやすさを重視しつつ、ユーザーを下げる印象の語を避ける
- 既存のMBTI名称とは重ならない独自名称とする

### 2.2 本書の位置づけ

- 本書の定義を、初期リリース時点の正式なタイプコード・タイプ名称とする
- 将来ブランド調整で表示名を見直す場合でも、タイプコードは原則として不変とする
- [specification.md](./specification.md) 5.4 の命名イメージは、本書の正式定義で置き換える

## 3. タイプコード設計

### 3.1 コード生成規則

タイプコードは4文字の英大文字で表現する。各文字は以下の診断軸に対応する。

1. 1文字目: 発言型 / 観察型
2. 2文字目: 事実重視 / 推理重視
3. 3文字目: 論理派 / 感情派
4. 4文字目: 計画型 / 即興型

例:

- `TFLP` = 発言型 / 事実重視 / 論理派 / 計画型
- `OREI` = 観察型 / 推理重視 / 感情派 / 即興型

### 3.2 軸と文字の対応表

| 診断軸 | 優勢側   | コード      | 対立側   | コード              |
| ------ | -------- | ----------- | -------- | ------------------- |
| 第1軸  | 発言型   | `T` = Talk  | 観察型   | `O` = Observe       |
| 第2軸  | 事実重視 | `F` = Fact  | 推理重視 | `R` = Reasoning     |
| 第3軸  | 論理派   | `L` = Logic | 感情派   | `E` = Emotion       |
| 第4軸  | 計画型   | `P` = Plan  | 即興型   | `I` = Improvisation |

### 3.3 実装ルール

- DB上の主キー相当は `typeCode` を利用する
- UIでは `typeCode` 単体ではなく、タイプ名称とセットで表示する
- SNSシェア文面では `タイプ名称（typeCode）` の形式を基本とする
- 将来的にコピーや説明文を調整しても、`typeCode` は変更しない

## 4. タイプ名称の命名方針

### 4.1 命名ルール

- 名称は「その人が卓でどう動くか」が伝わる職能名・役割名にする
- 名称だけで何となく強みがイメージできるようにする
- 推理ゲーム文脈に合う語彙を使い、過剰に中二的・攻撃的な表現は避ける
- 事実重視タイプには「論証」「証言」「痕跡」などの現場語彙を使う
- 推理重視タイプには「仮説」「裏筋」「物語」「心理」などの読解語彙を使う
- 発言型タイプには場を動かす印象の語を、観察型タイプには拾う・読む・設計する印象の語を優先する

### 4.2 名称構造の考え方

- 前半語: 何を頼りに盤面を見るか
- 後半語: 卓でどのような役回りを担いやすいか

例:

- `論証司令塔` = 証拠と論理を軸に、会話全体を主導するタイプ
- `余白読解者` = 発言されていない感情や含みを読み取るタイプ

| `OREP` | 観察 / 推理 / 感情 / 計画 | 感情の翻訳者 | 言葉にならない気持ちを、物語の輪郭に変える。   | 人物同士の感情の流れを読み、全体像を丁寧に組み立てる読解役     |

## 6. タイプマスタ初期定義

初期リリースのタイプマスタでは、少なくとも以下の項目を保持する。

| 項目名     | 必須 | 内容                             |
| ---------- | ---- | -------------------------------- |
| `typeCode` | 必須 | 本書で定義した4文字コード        |
| `typeName` | 必須 | 本書で定義した正式タイプ名称     |
| `tagline`  | 必須 | 一言キャッチコピー               |
| `summary`  | 必須 | 想定プレイ像をもとにした簡易説明 |
| `axis1`    | 必須 | 発言型 or 観察型                 |
| `axis2`    | 必須 | 事実重視 or 推理重視             |
| `axis3`    | 必須 | 論理派 or 感情派                 |
| `axis4`    | 必須 | 計画型 or 即興型                 |

## 7. 表示・運用ガイド

### 7.1 結果画面

- 結果画面の最上部は `タイプ名称` を主表示とする
- 補助表示として `typeCode` と `4軸バランス` を併記する
- 一言キャッチコピーはファーストビューに含める

推奨表示例:

```text
名探偵
TFLP | 発言型・事実重視・論理派・計画型
散らかった盤面ほど、この人の出番だ。
```

### 7.2 SNSシェア

- シェア文面では名称優先、コード補助とする
- タイプ名称のあとに、タイプの本質をひと言で体感させる一文を添える
- 例: `マダミスタイプは「名探偵（TFLP）」でした。盤面が散らかるほど光る——証拠を束ねて答えへ連れていく人らしい。`

### 7.3 将来改修時のルール

- キャッチコピーや詳細説明の改修は可能
- タイプ名称の微修正は可能だが、コードとの意味対応を崩さない
- 軸定義を変更する場合は、本書の全面改訂を前提とする
